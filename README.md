# Scentence

iOS-приложение для поиска парфюмерии по текстовым описаниям. Пользователь описывает желаемый аромат в свободной форме, а бэкенд на основе LLM возвращает подобранные рекомендации с оценкой релевантности, пирамидой нот и развёрнутыми описаниями.

## Возможности

- **Поиск на естественном языке** — опишите аромат произвольным текстом; модель на бэкенде подбирает и ранжирует результаты
- **Расширенные фильтры** — по ольфакторной группе, полу, типу продукта (EDP, EDT и др.), бренду, отдельным нотам и году выпуска
- **Карточка аромата** — полная пирамида нот, AI-теги, описание, похожие парфюмы
- **Избранное** — сохранение ароматов между сессиями
- **История поиска** — просмотр и повторный запуск прошлых запросов
- **Профиль** — аватар, имя, переключатель тёмной/светлой темы
- **Аутентификация по email** — вход через одноразовый код, без паролей

## Требования

- iOS 16.0+
- Xcode 15.0+
- Запущенный экземпляр бэкенда (API-адрес настраивается в `APIService.swift`)

## Быстрый старт

1. Клонируйте репозиторий:
   ```
   git clone https://github.com/sonches-k/scentence_app_frontend.git
   ```
2. Откройте `Scentence.xcodeproj` в Xcode.
3. В файле `Scentence/Core/Services/APIService.swift` укажите базовый URL вашего бэкенда:
   ```swift
   // DEBUG: по умолчанию localhost:8000
   // Release: задаётся через baseURL в APIService
   ```
4. Выберите симулятор или устройство и запустите схему **Scentence** (`⌘R`).

> **Примечание:** в DEBUG-сборке `Info.plist` разрешает произвольные сетевые запросы и локальный трафик. Для продакшна настройте ATS в соответствии с требованиями.

## Структура проекта

```
Scentence/
├── App/                    # Точка входа, глобальное состояние (AppState, AuthState), корневая навигация
├── Core/
│   ├── Models/             # Codable-модели данных (Perfume, Auth, Search, User)
│   ├── Services/           # APIService (сеть), KeychainService (хранение JWT)
│   └── Extensions/         # Цветовая тема, view-модификаторы, общие примитивы
├── Features/
│   ├── Auth/               # Флоу входа по email + OTP
│   ├── Search/             # Поисковый экран и лист фильтров
│   ├── Results/            # Список результатов с объяснением от LLM
│   ├── PerfumeDetail/      # Полный экран аромата, похожие, переключение избранного
│   ├── Favorites/          # Список сохранённых ароматов
│   ├── History/            # История поисковых запросов
│   └── Profile/            # Профиль пользователя и настройки
└── UI/Components/          # Переиспользуемые компоненты (PerfumeCard, RemoteImage, лоадеры)

ScentenceTests/
├── Mocks/                  # Mock-реализация APIServiceProtocol для тестов
├── Models/                 # Тесты декодирования/кодирования моделей
└── ViewModels/             # Тесты поведения ViewModel
```

## Архитектура

Приложение построено по паттерну **MVVM**:

- **Model** — `Codable`-структуры в `Core/Models/`, отображение snake_case → camelCase через `JSONDecoder`
- **Service** — `APIService` реализует `APIServiceProtocol` (протокол для подмены в тестах); `KeychainService` хранит JWT-токен
- **ViewModel** — `@MainActor final class … ObservableObject` с `@Published`-свойствами; все сетевые вызовы через async/await
- **View** — SwiftUI-представления, подключающие ViewModel через `@StateObject` / `@ObservedObject`
- **Глобальное состояние** — `AuthState` (аутентификация, текущий пользователь, токен) и `AppState` (выбранная вкладка, тёмная тема, межэкранная навигация) передаются через `@EnvironmentObject`

